## Implementation Notes

### Database Table: email_sequences
```sql
CREATE TABLE IF NOT EXISTS email_sequences (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_id INTEGER NOT NULL,
    customer_email TEXT NOT NULL,
    customer_name TEXT NOT NULL,
    sequence_type TEXT DEFAULT 'waitlist',
    email1_sent BOOLEAN DEFAULT 0,
    email1_sent_at TEXT,
    email2_sent BOOLEAN DEFAULT 0,
    email2_scheduled_at TEXT,
    email2_sent_at TEXT,
    email3_sent BOOLEAN DEFAULT 0,
    email3_scheduled_at TEXT,
    email3_sent_at TEXT,
    is_test_mode BOOLEAN DEFAULT 0,
    created_at TEXT NOT NULL,
    FOREIGN KEY(customer_id) REFERENCES customers(id)
);
```

### Test Mode
- Email contains `+test` (e.g., `tenmình+test@gmail.com`)
- All 3 emails sent immediately
- Emails still go to the correct inbox (email+test@domain.com works with Gmail, Outlook, etc.)

### Scheduler
- Uses APScheduler to run background tasks
- Every minute, checks for emails scheduled to be sent
- Sends Email 2 after 2 days (48 hours)
- Sends Email 3 after 1 more day (24 hours)

### Configuration
- API Key and sender email from `resend_config.txt`
- Sender domain: `hi@dozenko.io.vn` (from config)
- Payment link: `{payment_link}` placeholder for checkout page
